# E-commerce REST API

## Описание проекта
Backend-сервис для интернет-магазина, предоставляющий REST API для управления пользователями, каталогом товаров, корзиной и оформлением заказов. 

Проект спроектирован с учетом масштабируемости: применена логическая декомпозиция на независимые приложения (apps), бизнес-логика вынесена в отдельный сервисный слой (Service Layer), обеспечивается транзакционная целостность при работе с финансами и складом.

## Стек технологий
* **Язык:** Python 3.11+
* **Фреймворк:** Django 4+, Django REST Framework (DRF)
* **База данных:** PostgreSQL 16
* **Аутентификация:** JWT (djangorestframework-simplejwt)
* **Контейнеризация:** Docker, Docker Compose
* **Тестирование:** Pytest, pytest-django
* **Документация API:** drf-spectacular (Swagger UI)

## Архитектурные решения
* Изоляция бизнес-логики: контроллеры (views) отвечают только за маршрутизацию и HTTP-ответы. Вся работа с базой данных и расчеты инкапсулированы в `services.py`.
* Транзакции: оформление заказа (списание баланса, списание остатков со склада, очистка корзины) обернуто в `transaction.atomic()`. В случае ошибки на любом этапе происходит откат состояния БД.
* Оптимизация БД: устранена проблема N+1 запросов за счет использования `select_related` и `prefetch_related` в слоях агрегации данных.

## Инструкция по запуску

1. Склонируйте репозиторий:
git clone <url_репозитория>
cd <имя_папки>

Запустите сборку и старт контейнеров:
docker-compose up -d --build

Примените миграции базы данных:
docker-compose exec app python manage.py migrate

Создайте администратора для управления каталогом:
docker-compose exec app python manage.py createsuperuser

## Документация API
После успешного запуска сервиса интерактивная документация (Swagger) доступна по адресу:
http://127.0.0.1:8000/api/docs/

## Тесты
Для запуска тестов выполните команду:
docker-compose exec app pytest
